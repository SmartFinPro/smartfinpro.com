Übergangs-Regeln (bindend):
  • initial → ready → calculating → result ist der Normalpfad; calculating dauert 180–240 ms (CSS-Transition), unter prefers-reduced-motion 0 ms.
  • Example → Yours: beim ersten echten Input (tool_start); nie zurück zu Example innerhalb der Session.
  • Shared gilt nur bei gültigem Fragment-Payload; jede Bearbeitung wechselt auf Yours.
  • stale-data ist ein Overlay-Zustand von result (Ergebnis + Warnbanner), kein Ersatz.
  • insufficient-data und error zeigen NIE Zahlenwerte; ResultMiniBar (mobil) erscheint in diesen Zuständen nicht.
